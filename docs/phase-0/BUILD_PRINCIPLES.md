# Build Principles

## 1. Foundation Before Intelligence

We build deterministic systems first.

AI comes later.

## 2. Tests Protect the Product

Every critical rule must eventually have tests.

Priority test areas:

- chapter ordering
- concept ordering
- unlock rules
- locked concept blocking
- assessment scope
- admin permissions
- curriculum approval
- AI grounding

## 3. Admin Control Before Automation

No powerful AI automation should exist without admin visibility.

## 4. Draft Before Official

Parser output, AI output, and generated content begin as drafts.

Official learning content must be approved or compiled through trusted rules.

## 5. Observable by Default

Important events must be logged.

Examples:

- upload started
- upload completed
- parsing started
- parsing failed
- curriculum draft created
- curriculum approved
- lesson opened
- assessment submitted
- concept passed
- concept failed
- Ariel taught
- remediation started

## 6. Mobile First

Students are expected to use phones heavily.

Every learner-facing screen must work well on mobile.

## 7. Codex Later

Codex joins only after the architecture, naming, folder structure, rules, and early models are stable.
